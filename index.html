OWASP ThreatSpec

Code-driven threat modelling

Develop

Developers, QA, operations, and security engineers work closely together reviewing and annotating code with threat modeling metadata.

Parse

Use a parser to turn ThreatSpec code annotations into an intermediate json file.

Report

Run a report against one or more intermediate json files, even if they originate from different languages. Use the reports for further guide your threat modelling.

Try code-driven threat modelling

Write or edit JavaScript in the editor on the left and annotate it with ThreatSpec tags. Then run Parse & Report to generate the HTML report on the right. You can also view the intermediate JSON file.

Tag formats

  • @alias (boundary|component|threat) ALIAS to TEXT
  • @mitigates BOUNDARY:COMPONENT against THREAT with MITIGATION [(REFERENCE)]
  • @exposes BOUNDARY:COMPONENT to THREAT with EXPOSURE [(REFERENCE)]
  • @transfers THREAT to BOUNDARY:COMPONENT with TRANSFER [(REFERENCE)]
  • @accepts THREAT to BOUNDARY:COMPONENT with ACCEPTANCE [(REFERENCE)]
